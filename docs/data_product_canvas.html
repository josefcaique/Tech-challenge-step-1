Tech Challenge · Fase 1

Previsão de Churn · Telco Customer

Pipeline end-to-end de ML: do dado bruto ao modelo rastreado com MLflow. MLP PyTorch comparado com baselines Scikit-Learn.

🎯
Problema de Negócio
  • Operadora de telecom perdendo clientes em ritmo acelerado
  • Taxa de churn atual: 26,5% (1.869 de 7.043 clientes)
  • Identificar clientes em risco antes do cancelamento
  • Permitir ações proativas de retenção: desconto, ligação, oferta especial
Churn 26,5% Classificação Binária Retenção de Clientes
👤
Consumidores do Produto
  • Time de Retenção: lista diária de clientes em risco para ação
  • Diretoria: KPIs de churn evitado no dashboard executivo
  • CRM / Sistemas internos: integração via API FastAPI
  • Engenharia de ML: rastreabilidade via MLflow Registry
Time de Retenção Diretoria CRM MLOps

Modelos e Transformações
  • Pré-processamento: StandardScaler + OneHotEncoder
  • Baselines: DummyClassifier, LogisticRegression
  • Ensembles: RandomForest, XGBoost, CatBoost, DecisionTree, Ridge, MLP sklearn
  • Modelo principal: MLP PyTorch [128 → 64 → 32] com Dropout, Early Stopping e BCEWithLogitsLoss

Governança e Privacidade (LGPD)
  • customerID removido: identificador direto, não entra no modelo
  • gender excluído: irrelevante preditivamente e risco de viés
  • Dados anonimizados: nenhum PII sensível no conjunto de treino
  • Conformidade LGPD verificada antes da modelagem
  • SHA-256 do dataset registrado por run para rastreabilidade
LGPD Compliant Sem PII Anonimizado
☁️
SLA e Arquitetura de Deploy
  • Modo: Batch, processamento noturno com lista diária de risco
  • API: FastAPI com /, /health, /model e /predict via Pydantic
  • Latência: batch tolerante a delay; real-time disponível via POST /predict
  • Reprodutibilidade: seeds fixados, pyproject.toml, Poetry lock
  • Deploy local: Docker Compose (Postgres + MinIO + MLflow + FastAPI) na porta 8000
  • Deploy AWS Lambda: docker/lambda-api/Dockerfile + Mangum como adapter ASGI → Lambda
  • Modelo carregado: MODEL_URI=models:/churn_prediction_pipeline/latest via MLflow Registry
Batch + API FastAPI AWS Lambda Docker Compose Reprodutível

Rastreabilidade via MLflow
  • Tracking: parâmetros, métricas e artefatos por run
  • Model Registry: versionamento com alias baseline e challenger
  • Version tags: stage, model_family e framework por versão
  • Descrições: centralizadas em registry.py, desempacotadas nos notebooks
  • Dataset hash: SHA-256 registrado para reprodutibilidade
